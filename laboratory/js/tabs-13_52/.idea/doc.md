1. Как должен выглядеть HTML (Архитектура)

Чтобы табы работали, нам нужно разделить интерфейс на две независимые зоны:

    Навигация (Пульт): Группа кнопок.

    Контент (Экран): Группа блоков с информацией, из которых в один момент времени виден только один.

2. Как связать кнопку с контентом? (Связующее звено)

Есть два основных способа "склеить" кнопку №2 с блоком №2:
Способ А: Data-атрибуты (Профессиональный)

Мы вешаем на кнопку "бирку" data-target="tab-2". А блоку контента даем id="tab-2".

    Логика: Когда мы кликаем на кнопку, JS считывает значение из data-target и ищет в документе элемент с таким же ID. Это очень надежно, потому что порядок кнопок в HTML может измениться, но связь по ID останется.

Способ Б: По индексу (Простой)

Мы знаем, что кнопка "Характеристики" — вторая по счету (индекс 1 в массиве). И блок с характеристиками тоже второй по счету в своем контейнере.

    Логика: При клике на кнопку №1 мы просто открываем блок №1. Это быстро писать, но хрупко (если добавить лишнюю кнопку, индексы поплывут).

3. Какое состояние подготовить в CSS?

Тут всё так же, как в Аккордеоне, но для блоков контента:

    .tab-pane (по умолчанию): display: none;. Все блоки скрыты.

    .tab-pane.active: display: block;. Виден только тот, у которого есть этот класс.

    .tab-btn.active: Можно подсветить саму кнопку (например, изменить цвет фона), чтобы пользователь видел, какая вкладка сейчас выбрана.
